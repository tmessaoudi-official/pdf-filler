Accessibility Statement

Partially conformant — WCAG 2.1 AA

We are committed to making PDFturbo accessible to as many users as possible, including those using assistive technologies. Our target is WCAG 2.1 Level AA conformance.

What we have done

  • All toolbar buttons have accessible names via aria-label and/or visible text.
  • Tool-mode toggle buttons expose their state via aria-pressed.
  • All modal dialogs use role="dialog", aria-modal="true", and aria-labelledby.
  • Focus is trapped inside open modals (Tab / Shift+Tab cycles within) and returns to the trigger on close.
  • The find-bar result count is announced via aria-live="polite".
  • The app supports keyboard-only navigation (all tools accessible via keyboard shortcuts).
  • The UI supports right-to-left layouts for Arabic.
  • Color contrast meets WCAG AA for all interactive controls.

Known limitations

  • PDF text layer: the selectable text layer over PDF content is not yet implemented. This means screen readers cannot access native PDF text (only annotated text elements added by the user).
  • The drawing canvas for signatures and freehand drawing is not fully accessible to screen-reader users.

Feedback and contact

If you encounter accessibility barriers, please contact us at: takieddine.messaoudi.official@gmail.com

We aim to respond within 10 business days.

Déclaration d'accessibilité

Partiellement conforme — WCAG 2.1 AA

Nous nous engageons à rendre PDFturbo accessible au plus grand nombre, y compris aux utilisateurs ayant recours à des technologies d'assistance. Notre objectif est la conformité WCAG 2.1 Niveau AA.

Ce que nous avons mis en place

  • Tous les boutons de la barre d'outils disposent d'un nom accessible via aria-label et/ou du texte visible.
  • Les boutons de mode basculent leur état via aria-pressed.
  • Toutes les boîtes de dialogue utilisent role="dialog", aria-modal="true" et aria-labelledby.
  • Le focus est piégé dans les modales ouvertes (Tab / Maj+Tab cycles interne) et retourne au déclencheur à la fermeture.
  • Le nombre de résultats de recherche est annoncé via aria-live="polite".
  • L'application supporte la navigation au clavier uniquement.
  • L'interface supporte les mises en page de droite à gauche (arabe).
  • Le contraste des couleurs respecte le niveau AA pour tous les contrôles interactifs.

Limitations connues

  • Couche de texte PDF : la couche de texte sélectionnable sur le contenu PDF n'est pas encore implémentée. Les lecteurs d'écran ne peuvent donc pas accéder au texte natif du PDF.
  • Le canvas de dessin (signatures et tracé libre) n'est pas entièrement accessible aux utilisateurs de lecteurs d'écran.

Retours et contact

Si vous rencontrez des obstacles à l'accessibilité, contactez-nous à : takieddine.messaoudi.official@gmail.com

Nous visons à répondre dans un délai de 10 jours ouvrés.

مطابقة جزئية — WCAG 2.1 AA

نلتزم بجعل PDFturbo في متناول أكبر عدد ممكن من المستخدمين، بمن فيهم من يستخدمون التقنيات المساعدة. هدفنا هو الامتثال لمعيار WCAG 2.1 المستوى AA.

ما تم تطبيقه

  • جميع أزرار شريط الأدوات لها أسماء يمكن الوصول إليها عبر aria-label أو نص مرئي.
  • أزرار وضع الأدوات تُظهر حالتها عبر aria-pressed.
  • جميع مربعات الحوار تستخدم role="dialog" وaria-modal="true" وaria-labelledby.
  • يُحبس التركيز داخل مربعات الحوار المفتوحة ويعود إلى العنصر المُشغِّل عند الإغلاق.
  • عدد نتائج البحث يُعلَن عبر aria-live="polite".
  • التطبيق يدعم التنقل بالكيبورد فقط.
  • الواجهة تدعم تخطيطات من اليمين إلى اليسار للغة العربية.
  • تباين الألوان يلبي مستوى AA لجميع عناصر التحكم التفاعلية.

القيود المعروفة

  • طبقة نص PDF: طبقة النص القابلة للتحديد فوق محتوى PDF لم تُطبَّق بعد. وهذا يعني أن قارئات الشاشة لا تستطيع الوصول إلى النص الأصلي لملف PDF.
  • لوحة الرسم (التوقيعات والرسم الحر) ليست متاحة بالكامل لمستخدمي قارئات الشاشة.
